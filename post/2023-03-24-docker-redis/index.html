docker 搭建Redis 集群

环境准备

  1. 需要三台机器

  2. 每台机器都需要安装docker

    yum -y install docker

  3. 内核开启路由转发

    echo " net.ipv4.ip_forward = 1" >> /etc/sysctl.conf

    sysctl -p

集群规划

下面步骤中配置文件中的node 请自行替换为你宿主机的IP

例如

node1 => 192.168.1.1

node2 => 192.168.1.2

node3 => 192.168.1.3

后续文档将不再赘述, 请注意修改 ! !

一共三台机器, 每台启用两个实例, 组成三节点1副本集群.

node1: 6379,6380

node2: 6381,6382

node2: 6383,6384

所有节点拉取镜像

1

docker pull redis:5.0

mkdir -p /data/redis-6379

cat >/data/redis-6379/redis.conf<<'EOF'
### 集群配置
port 6379
protected-mode no
cluster-enabled yes
cluster-config-file nodes.conf
cluster-node-timeout 5000
# 这里填写的是你机器对应的ip
cluster-announce-ip 192.168.1.1
cluster-announce-port 6379
cluster-announce-bus-port 16379
appendonly yes
########################
EOF



 docker run -d --net host --privileged=true \
  -v /data/redis-6379/redis.conf:/usr/local/etc/redis/redis.conf \
  -v /data/redis-6379:/data \
  --restart always --name redis-6379 \
  redis:5.0 redis-server /usr/local/etc/redis/redis.conf; \

创建集群

进入随便一个node 进入redis容器

我这里直接进入node1 实例一

1
2
3
4
5
6
7

# 进入容器
docker exec -ti redis-6379 bash


# 创建集群, 请注意替换为自己的IP(
# 提示输入yes)
redis-cli --cluster create  192.168.1.1:6379 192.168.1.1:6380 192.168.1.2:6381 192.168.1.2:6382 192.168.1.3:6383 192.168.1.3:6384 --cluster-replicas 1

查看集群信息

1

redis-cli -c -p 6379 cluster info  # -c 用集群方式登录 -p 指定端口
